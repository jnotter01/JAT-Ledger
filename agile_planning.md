## 1. Product Vision Statement

Johnny Apple Trees Ledger is a web-based application designed for small rental property owners who need a simple way to track their financial data. The system replaces scattered spreadsheets and manual tracking with a centralized platform for managing properties, income, and expenses.

The product provides clear financial visibility by allowing users to record transactions and view summaries of profit and loss. This helps property owners make better business decisions, stay organized, and prepare for taxes more efficiently.

## 3. User Stories

1. As a property owner, I want to add and manage properties so that I can keep track of my rental units.

2. As a property owner, I want to record income so that I can track rent payments.

3. As a property owner, I want to log expenses so that I can monitor costs.

4. As a property owner, I want to view all transactions in one place so that I can understand my financial activity.

5. As a property owner, I want to see a summary of profit and loss so that I can evaluate my business performance.

## 4. Sprint Plan (Sprint 1)

**Sprint Duration:** 1 week

**Sprint Goal:**  
Establish the foundation of the application by setting up the project and implementing core data structures.

**Selected Backlog Items:**
- Project Setup
- Database Models
- Admin Panel Setup
- Property CRUD

**Justification:**
These tasks provide the foundation for the entire system. Without the project structure and data models, other features cannot be built. Property management is prioritized first because all transactions depend on properties being defined.

## 5. Team Roles and Responsibilities

Since this is a solo project, all roles will be performed by a single developer.

- **Scrum Master (Coordination):** Responsible for planning tasks, tracking progress, and maintaining organization.
- **Product Owner (Priorities):** Responsible for defining features and prioritizing backlog items.
- **Developer (Implementation):** Responsible for coding, testing, and debugging the application.

Although all roles are handled by one person, responsibilities will be managed separately to maintain structure and organization.

## 6. Reflection

The most challenging part of Agile for this project will be maintaining discipline in planning and iteration while working alone. Without a team, it's easy to skip structured processes and jump straight into coding.

Agile practices will help reduce project failure by breaking the work into smaller, manageable pieces and focusing on incremental progress. By using a backlog and sprint planning, the project remains organized and adaptable to changes.

To simulate feedback and iteration, progress will be reviewed regularly, and features will be tested after each sprint to ensure they meet the intended goals.
